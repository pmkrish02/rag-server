# RAG Server

A Retrieval Augmented Generation server built in Go with Weaviate vector database and Google Gemini API. Ingest documents, store them as vector embeddings, and ask questions answered using retrieved context.

## What It Does

1. **Ingest**: Upload documents → embed with Gemini → store text + vectors in Weaviate
2. **Query**: Ask a question → embed it → find similar documents → Gemini generates an answer grounded in your data

## Tech Stack

- **Go** — HTTP server, request handling
- **Weaviate** — vector database for semantic storage and search
- **Gemini API** — `gemini-embedding-001` for embeddings, `gemini-2.0-flash` for answer generation
- **GraphQL** — Weaviate's query interface for vector similarity search

## Endpoints

### `POST /documents/`
Ingest documents into the vector database.
```bash
curl -X POST http://localhost:8000/documents/ \
  -H "Content-Type: application/json" \
  -d '{
    "documents": [
      {"text": "Photosynthesis is the process by which plants convert sunlight into energy"},
      {"text": "The mitochondria is the powerhouse of the cell"},
      {"text": "DNA carries genetic instructions for development and functioning"}
    ]
  }'
```
Response: `stored 3 documents`

### `POST /query/`
Ask a question — the server retrieves relevant context and generates an answer.
```bash
curl -X POST http://localhost:8000/query/ \
  -H "Content-Type: application/json" \
  -d '{"user_query": "How do plants get energy?"}'
```
Response: A Gemini-generated answer grounded in the documents you ingested.

## How to Run

1. Set up a [Weaviate Cloud](https://console.weaviate.cloud) free sandbox instance.

2. Set environment variables:
```bash
export WEAVIATE_HOSTNAME="your-cluster.weaviate.network"
export WEAVIATE_API_KEY="your-weaviate-api-key"
export GOOGLE_API_KEY="your-gemini-api-key"
```

3. Install dependencies and run:
```bash
go mod tidy
go run main.go
```

Server starts on `:8000`.

## Architecture

```
Ingest Flow (/documents/):
    JSON payload
        → Decode documents
        → Batch embed via Gemini (gemini-embedding-001)
        → Store text + vector in Weaviate

Query Flow (/query/):
    User question
        → Embed question via Gemini
        → Vector similarity search in Weaviate (top 2 results)
        → Build prompt: context + question
        → Gemini generates answer from context
        → Return answer as JSON
```

## Key Design Decisions

- **Bring-your-own-vectors** — embeddings generated by Gemini, not Weaviate's built-in vectorizer, giving full control over the embedding model
- **Per-request context** — handlers use `r.Context()` instead of a shared context, enabling safe concurrent request handling and proper cancellation
- **Idempotent startup** — checks if the Weaviate collection exists before creating, preventing data loss on server restart
- **Struct methods over closures** — handlers are methods on `ragServer` struct, giving access to both Weaviate and Gemini clients
- **Batch embedding** — all documents embedded in a single API call rather than one-by-one
